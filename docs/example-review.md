# Example: what a posted review looks like

When the reviewer runs on a pull request, it posts one **review** containing
inline comments on the exact changed lines plus an aggregated summary. A mock
of that output on GitHub:

---

### Inline comment on `app/charge.py` line 42

> **🛑 CRITICAL · Security** — Hardcoded Stripe secret
>
> A live API key is committed in source. Rotate it immediately and load it from
> the environment instead.
>
> ````suggestion
> stripe.api_key = os.environ["STRIPE_KEY"]
> ````

### Inline comment on `app/charge.py` line 88

> **🔴 HIGH · Correctness** — Refund amount can go negative
>
> There is no lower bound on the refund value, so a malformed request can invert
> the original charge.

---

### Summary comment (review body)

> ## 🤖 AI PR Review
>
> **🔴 Changes requested**
>
> | Severity | Count |
> | --- | --- |
> | 🛑 critical | 1 |
> | 🔴 high | 1 |
> | 🟡 low | 1 |
>
> **3** finding(s) total.
>
> <sub>Generated by the AI GitHub PR Reviewer.</sub>

The review **event** is derived from the verdict: findings at or above the
configured `block_severity` produce **Request changes**, other findings produce
**Comment**, and a clean diff produces **Approve**.
